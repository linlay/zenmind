Runner

Agent Platform Runner

镜像产品。由 ZenMind docker compose 统一管理，并暴露 `/api/ap/*`。

Runner 关闭时，Gateway 不再代理 `/api/ap/*`，内部服务也不会使用容器内 runner。

镜像容器内部固定监听 `8080`，这里只维护宿主机映射端口。

兼容旧配置保留的派生值，保存时会按 Runner 宿主机端口自动写回。

Admin

管理端

后台管理与 App 登录主凭据入口。

管理端关闭时不会显示具体配置。打开左侧开关后，再填写登录密码和高级项。

页面输入的是明文，保存到 JSON 时只会写入 bcrypt。

输入明文后会立即转换成 bcrypt，JSON 只保存 hash。

用于 App 登录主凭据，页面只暂存明文，导出时不会写入原文。

输入明文后会立即转换成 bcrypt，JSON 只保存 hash。

控制 `/admin` 与 `/admin/api` 是否对外暴露。

低频配置，默认端口通常不需要改动。

Pan

网盘

文件访问、网页登录与 App 入口配置。

网盘关闭时只保留分组开关。开启后先填登录密码，再按需展开高级选项。

网页登录密码，页面会立即转换成 bcrypt，导出 JSON 不保留明文。

输入明文后会立即转换成 bcrypt，JSON 只保存 hash。

控制 `/pan/` 浏览器入口是否开放。

低频配置，通常沿用默认值。

首次启用网盘时会自动生成；`/apppan` 会随网盘服务启用而开放，不再单独配置。

Term

终端

终端浏览器与 App 访问入口配置。

终端关闭时不显示具体字段。开启后默认只维护登录密码。

终端网页登录密码，页面会立即转换成 bcrypt，导出 JSON 不保留明文。

输入明文后会立即转换成 bcrypt，JSON 只保存 hash。

控制 `/term/` 浏览器入口是否开放。

低频配置，通常不需要调整；`/appterm` 会随终端服务启用而开放。

Container Hub

Container Hub

宿主机程序，由 setup 负责启动、停止与状态查看。

Container Hub 关闭时，setup 不会启动宿主机进程，runner 的 container-hub 配置也会保持 disabled。

宿主机程序默认监听 `127.0.0.1`，runner 容器通过 `host.docker.internal` 访问它。

默认本地回环监听可以为空；如果后续改成非本地地址，再按需填写。
